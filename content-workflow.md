# 콘텐츠 자동화 워크플로우 — 포항 기업혁신파크

> 게시판에 올린 공지·소식 원고 하나를 여러 채널 콘텐츠로 재활용하는 운영 흐름입니다.
> 핵심 원칙: 자동화로 효율을 높이되, 신뢰에 직결되는 정보는 사람이 반드시 검토합니다.

## 전체 흐름 한눈에 보기

```
[1] 원고 수집(게시판/Notion)
        ↓
[2] AI 초안 생성(블로그·SNS·보도자료)
        ↓
[3] 🔴 사람 검토 (필수) — 입주기업 이름·성과 수치 직접 확인
        ↓
[4] 채널별 발행
        ↓
[5] 내부 링크·사이트맵 갱신
```

## 1. 원고 수집
- 입력 소스: 공지·소식 게시판 원고, Notion에 모아둔 초안
- 원고 1건마다: 제목, 본문, 카테고리(공지/소식/모집), 관련 입주기업, 첨부 이미지 정리

## 2. AI 초안 생성 (재활용 포맷)
하나의 원고에서 아래 4종 초안을 자동 생성합니다.
| 결과물 | 용도 | 길이/톤 |
|--------|------|---------|
| 블로그 글 | 사이트 블로그/소식 | 길고 상세, 신뢰 톤 |
| 인스타그램 캡션 | SNS 도달 | 짧고 친근, 해시태그 |
| 링크드인 포스트 | 기업·파트너 대상 | 전문적, 성과 중심 |
| 보도자료 초안 | 언론 배포 | 격식체, 사실 위주 |
| 내부 링크 목록 | SEO/회유 동선 | 관련 페이지 3~5개 |

## 3. 🔴 사람 검토 (필수 승인 단계)
> 이 단계는 건너뛸 수 없습니다. 자동 발행 금지.

**반드시 직접 확인하는 항목 (학습자 지정):**
- ✅ **입주기업 이름** — 정확한 상호/표기(법인명, 띄어쓰기, 영문명) 확인
- ✅ **성과 수치** — 매출·투자·고용 등 숫자가 원자료와 일치하는지 확인, 출처 확인
- ✅ 모집 일정·자격 등 공식 정보의 정확성
- ✅ 보도자료 톤이 과장되지 않았는지

검토 통과 시에만 다음 단계로 넘어갑니다. 수정 필요 시 2단계로 돌아가 재생성.

## 4. 채널별 발행
- 블로그: 사이트 게시판/블로그에 등록 (관리자 권한)
- SNS: 인스타그램·링크드인 예약 발행
- 보도자료: 담당자 최종 확인 후 배포
- 이미지: 관리자가 alt 텍스트와 함께 업로드

## 5. 내부 링크 · 사이트맵 갱신
- 새 글에서 관련 페이지(입주 안내·사례 등)로 내부 링크 연결
- sitemap.xml에 새 URL 추가, 검색엔진에 제출

- [ ] title / meta description 작성됨
- [ ] JSON-LD 스키마 마크업 포함
- [ ] sitemap.xml에 새 페이지 반영
- [ ] robots.txt의 Sitemap 경로 정상

- [ ] example.com → 실제 도메인 교체 완료
- [ ] 모바일 화면 깨짐 없는지 확인
- [ ] 외부/내부 링크 클릭 검수
- [ ] 관리자 외 사용자에게 게시판 수정 권한 노출 안 됨
